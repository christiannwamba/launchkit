## Prep

### Locked Stack

| Job | Tool | Why |
|-----|------|-----|
| Present | Next.js (App Router) | Industry-standard React framework |
| Store | Convex | Real-time backend, no SQL, free tier |
| Secure | WorkOS via Convex AuthKit | Auth bundled into Convex — one fewer integration |
| Process | Your code (generated by ChatGPT) | Business logic lives in Convex functions |
| Transmit | Convex (real-time) + Vercel (hosting) | Convex handles sync, Vercel serves the frontend |

### What Costs Money

ChatGPT Plus — $20/month. Everything else (Convex, WorkOS, Vercel, GitHub) is free tier.

### Scene 6 — Project Setup & Theming

**Type:** Screen recording + ChatGPT desktop app

**Goal:** Go from nothing to a running, themed project that's already on GitHub — in one scene. A custom skill handles everything: dependency checks, project creation, GitHub push, skill installation (local to the project), shadcn theming, component install, and agent.md creation. The only interactive moments are GitHub CLI auth, Convex GitHub sign-in, and choosing a theme color.

**Preconditions:**
- ChatGPT desktop app (or Claude Code) open and ready
- Terminal open

**Say before the prompt:** "The only thing that costs money here is ChatGPT Plus — $20 a month. Everything else we're about to use is free."

**Step 0: Create the folder and install the setup skill**

Say: "First, let's create a folder for our project and install a setup skill I've prepared. This skill knows exactly how to bootstrap everything so we don't have to do it piece by piece."

Paste this prompt into the agent:

```
Create a folder called "feedback-board" and go into it. Then run these commands to install two skills I want to use:

npx skills add christiannwamba/launchkit --skill setup-feedback-board -y
npx skills add christiannwamba/launchkit --skill plan-product -y

The first one sets up the entire project. The second one helps us plan what to build. Let me know when they're done.
```

After the skill is installed, **start a new chat** so the agent picks up the newly installed skill.

**Step 1: Paste this prompt into the new chat:**

```
I want to build a customer feedback board — a website where people can see feedback others have submitted, sign in to submit their own feedback, and vote on ideas they like.

Run the setup-feedback-board skill to set up the project for me. It will check what tools I need, install anything that's missing, create the project, push it to GitHub, install the coding skills, and set up the theme.

When it asks me to sign in to anything, I'll do that. When it asks me to pick a theme color, I'll go to the link and choose one.
```

**What happens on screen:**

The agent follows the skill steps automatically. The user only intervenes when something interactive is needed (browser sign-ins, theme choice).

1. **Dependency check + install** — verifies node, npm, git, gh, npx. If anything is missing, the agent installs it automatically (e.g., `brew install gh`). No user action needed unless Node.js is completely missing (rare).

2. **GitHub auth** — if `gh` is not authenticated, the agent runs `gh auth login`. This opens the browser.
   - Say: "It needs me to sign into GitHub. This lets us save our code and is how we'll deploy later."
   - Show B-roll of the GitHub signup docs page (`https://docs.github.com/en/get-started/start-your-journey/creating-an-account-on-github`) briefly: "If you don't have a GitHub account, go to github.com — it's free."

3. **Project creation** — runs `npm create convex@latest . -- -t nextjs-authkit` in the current directory (non-interactive, no menus).
   - **Convex sign-in moment:** The browser opens for GitHub sign-in. Say: "It's asking me to sign in with GitHub — this creates my free Convex account automatically. And it also sets up WorkOS for sign-in. One sign-in, two services."

4. **Push to GitHub** — the agent runs `gh repo create feedback-board --public --source=. --push`. Say: "Our code is already on GitHub. Every change from here gets saved there."

5. **Agent skills install** — installs seven skills **locally** into the project (not global — they stay in `.claude/skills/`): frontend-design, web-design-guidelines, vercel-react-best-practices, convex, convex-helpers-guide, workos, and shadcn.

6. **Theme pause** — the agent stops and asks you to pick a theme.
   - Open `https://ui.shadcn.com/themes` on screen
   - Say: "Before we start building features, let's make this look like ours. I'm going to pick [your color]."
   - Give the agent the CLI command or CSS variables — it applies the theme for you.

7. **Component install** — installs shadcn button, card, input, textarea, badge, dialog (non-interactive).

8. **agent.md created** — the agent writes `agent.md` at the project root. This tells any AI agent working on the project how to run it, how to validate changes, and what skills are available. Say: "This file is like a cheat sheet for the AI — it tells it how the project works so it doesn't have to guess."

9. **Validation** — runs `npx convex dev --once` to confirm everything compiles.

10. **Dev server starts** — `npm run dev` launches. `localhost:3000` loads with your theme.

11. **Handoff** — the agent mentions `npx vercel login` for later (deployment scene). No need to run it now.

**Verification:**
- Browser shows the themed Next.js page at `localhost:3000`
- Terminal shows no errors
- Convex dashboard at `dashboard.convex.dev` shows the project
- GitHub repo exists at `github.com/YOUR_USERNAME/feedback-board`
- `agent.md` exists at the project root
- Theme colors are visible on the page

**Recovery:**
- If a dependency install fails (e.g., `brew` not found): the skill will tell the user what to install manually. Follow its instructions, then tell the agent to continue.
- If Convex login fails: make sure GitHub is accessible in the browser and you can log in there.
- If `gh repo create` fails: "I got an error creating the GitHub repo: [paste]. Fix it."
- If the theme doesn't apply: "The theme colors aren't showing. Check that the CSS variables were added to the global styles file and that shadcn was initialized."
- If `npx convex dev --once` fails: "The validation step found errors. Here's the output: [paste]. Fix it."

### Scene 7 — Plan the Build

**Type:** Screen recording + ChatGPT desktop app

**Goal:** Generate a step-by-step build plan where every step produces something visible and verifiable. The plan-product skill drives the format — prompts are beginner-friendly, steps are product-first.

**Preconditions:**
- Project running with theme applied (Scene 6)
- ChatGPT desktop app open, model set to **ChatGPT 5.0**

**Say:** "Before we write any code, we're going to plan. I want every step to build something I can actually see working — no guessing, no blind faith."

**Paste this prompt into ChatGPT:**

```
I'm building a customer feedback board. Here's what the finished product should do:

1. Anyone can visit the site and see a list of feedback that other people have submitted
2. Anyone can click on a feedback item to see the full details
3. If you sign in, you can submit new feedback with a title and description
4. If you sign in, you can vote on feedback you like — but only once per item (clicking again removes your vote)
5. Sign in and sign out are already set up through WorkOS AuthKit

Use the plan-product skill to create a build plan in docs/feedback-board-plan.md. Each step should build one piece of the product that I can immediately see and check — in my browser or in the Convex dashboard. No steps where I just have to trust that something worked.
```

**What happens on screen:**
- Paste the prompt
- ChatGPT generates `docs/feedback-board-plan.md` following the plan-product skill format
- Open `docs/feedback-board-plan.md` in the editor, briefly scroll through
- Point out the structure: each step has what we're building, a prompt to paste, how to verify, and what to do if it breaks

**Say:** "Each step in this plan builds something real. It even gives us the prompts we'll paste — written so anyone can understand them, no coding jargon."

**Verification:**
- `docs/feedback-board-plan.md` exists in the project
- Each step has: what we're building, a prompt, verification instructions, and recovery
- Steps are ordered product-first (see results early), not layer-first
- Prompts describe behavior, not implementation

**Recovery:**
- If the plan is too technical: "The prompts in the plan mention file paths and code details. Rewrite them so they describe what the feature does, not how to build it. Someone with no coding experience should be able to read each prompt and understand it."
- If the plan is too vague: "Some steps don't tell me what I should see on my screen. For every step, tell me exactly what page to open and what should be there. Update the plan."

### Scene 8 — Build the Product

**Type:** Screen recording + ChatGPT desktop app

**Goal:** Execute the plan step by step. Each step: paste the prompt from docs/feedback-board-plan.md, let the agent build it, verify it works, move on.

**Preconditions:**
- `docs/feedback-board-plan.md` exists (Scene 7)
- Dev server running (`npm run dev`)
- ChatGPT desktop app open, switch model to **ChatGPT 5.4**

**Say:** "Now we build. The plan has everything — the prompts, the verification steps, even what to do if something breaks. We just follow it."

**How this scene works:**

This is not scripted step by step in the runbook. The docs/feedback-board-plan.md drives everything. For each step in the plan:

1. **Read the step** — look at what we're building and what the prompt says
2. **Paste the prompt** from docs/feedback-board-plan.md into ChatGPT
3. **Let the agent work** — it writes the code
4. **Validate** — run `npx convex dev --once` to check the backend compiles
5. **Verify** — follow the verification instructions in docs/feedback-board-plan.md (open a page, click something, check the dashboard)
6. **If it fails** — use the recovery template in docs/feedback-board-plan.md: describe what you see, paste the error, ask the agent to fix it
7. **Move to the next step**

**What to show on screen for each step:**
- The prompt being pasted
- The agent working
- The browser or Convex dashboard showing the result
- A brief verbal confirmation: "Step N done. Here's what we got: [describe]."

**Key moments to narrate:**

The plan will likely include steps for:
- Setting up data and seeing it in the Convex dashboard
- Showing feedback on the homepage
- Building the detail page
- Wiring up sign-in / sign-out
- Building the submission form
- Adding voting

For each of these, use the talking points from the outline:
- Data + list: "Every card is pulling live data from Convex."
- Auth: "We didn't build any sign-in logic — WorkOS and Convex handle that. We just connected the button."
- Submission: "I submit feedback and it appears instantly — no page refresh. That's Convex doing real-time sync."
- Voting: "Last feature. Voting works, it persists, and you can't double-vote. That's the entire product — built with ChatGPT prompts."

**Recovery:**
- If any step fails, follow the recovery template in docs/feedback-board-plan.md
- If a step fails three times: cut the recording, reset to the last working state (`git stash` or `git checkout`), and restart that step fresh
- Never move to the next step until verification passes

### Scene 9 — Ship

**Type:** Screen recording + ChatGPT desktop app

**Goal:** Deploy the app to production. The code is already on GitHub (pushed during setup). We just need Vercel auth and deployment.

**Preconditions:**
- All features work locally (Scene 8)
- Code already on GitHub (pushed in Scene 6)
- `gh` already authenticated (Scene 6)

**Say:** "The app works locally and our code is already on GitHub. Now let's put it on the internet."

**Step 1: Authenticate with Vercel (interactive)**

Say: "We need to connect to Vercel — this is what puts our site on the internet."

Show B-roll of the Vercel getting started page (`https://vercel.com/docs/getting-started-with-vercel`) briefly while explaining: "Sign up at vercel.com with your GitHub account — it's free."

Run in terminal:
```bash
npx vercel login
```

This opens the browser for authorization. Wait for it to complete.

**Step 2: Paste this prompt into ChatGPT:**

```
The feedback board is working on my computer and the code is already on GitHub. Now I want to put it on the internet so anyone can visit it.

I've already signed into Vercel from my terminal. Here's what I need you to do:

1. Push the latest code to GitHub (the repo already exists)
2. Deploy the website to Vercel and connect it to the GitHub repository
3. Set up the live database and sign-in so everything works on the internet, not just on my computer — look at the Convex dashboard settings and the existing environment variables to figure out what the live site needs

When there's a command you can run for me, just run it. If something needs me to open a page in my browser or click a button on a website, tell me exactly what to do.

When we're done, I should be able to open a URL and see my feedback board live with sign-in and voting working.
```

**What happens on screen:**
- Paste the prompt — ChatGPT pushes latest code and handles Vercel deployment non-interactively
- ChatGPT may ask you to open the Vercel dashboard or Convex dashboard to configure environment variables — follow those instructions on screen
- Open the live URL, verify everything works

**Verification:**
- Live URL loads the feedback board
- Auth works (sign in → name shows → sign out)
- Submission works (new item appears)
- Voting works (count changes, persists)

**Say:** "Building is exciting, but shipping is the part that changes your life. This app is live. Someone could use it today."

**Recovery:**
- If the deploy fails: "Vercel says the build failed. Here's the error from the build log: [paste]. What's wrong?"
- If sign-in doesn't work on the live site: "The app loads but when I click sign in, it gives an error. It works on my computer but not on the live site. How do I fix the sign-in settings for the live URL?"
- If the live site shows no data: "The site loads but there's no feedback data. On my computer I can see the data. How do I get the live site connected to the right database?"

## Recovery

### Recovery Rules

- **If a scene fails once:** Re-prompt ChatGPT with the error. Most failures are fixable in one follow-up.
- **If a scene fails twice:** Stop. Check the Convex dashboard logs and browser console. Fix the specific issue before re-prompting.
- **If a scene fails three times:** Cut the recording. Reset to the last working state (`git stash` or `git checkout`), re-read the docs/feedback-board-plan.md step, and start the scene fresh.
- **Never ship a scene where the verification step didn't pass.**

### Global Reset (Reset Data, Keep Code)

If the database gets into a bad state:

```bash
# Go to dashboard.convex.dev → Data → feedback table → delete all rows manually
# Then re-seed:
npx convex run seed
```

### Full Reset (Start Over)

If the project is too broken to continue:

```bash
# Save your plan first
cp docs/feedback-board-plan.md ~/Desktop/plan-backup.md

# Delete the project
cd ..
rm -rf feedback-board

# Start fresh from Scene 6
npm create convex@latest feedback-board -- -t nextjs-authkit
```

Then re-run the setup skill (Scene 6), re-generate the plan (Scene 7), and follow the plan (Scene 8).
